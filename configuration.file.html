فایل پیکربندی

فایل پیکربندی (php.ini) زمانی که برنامه اجرا می‌شود، خوانده می‌شود. برای نسخه های ماژول PHP این فقط یک بار زمانی که وب سرور راه اندازی می شود اتفاق می افتد. برای نسخه‌های CGI و CLI، این اتفاق در هر فراخوانی رخ می‌دهد.

php.ini : در این آردس ها یافت می‌شود (به ترتیب)

  • مکان خاص ماژول SAPI (دستورالعمل PHPIniDir در آپاچی 2، گزینه خط فرمان -c در CGI و CLI)
  • متغیر طبیعی PHPRC
  • آدرس فایل php.ini را می توان برای نسخه های مختلف PHP تنظیم کرد. ریشه کلیدهای رجیستری به 32 یا 64 بیتی سیستم عامل و PHP نصب شده بستگی دارد. برای PHP 32 بیتی در سیستم عامل 32 بیتی یا PHP 64 بیتی در سیستم عامل 64 بیتی، از [(HKEY_LOCAL_MACHINE\SOFTWARE\PHP] برای نسخه 32 بیتی PHP در سیستم عامل 64-بیتی استفاده کنید. بیت OS از [HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\PHP]] استفاده می کند. برای نصب بیتی مشابه کلیدهای رجیستری زیر را انجام دهید به ترتیب بررسی می شوند: [HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x.y.z], [HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x.y] و [HKEY_LOCAL_MACHINE\SOFTWARE\PHP\x], که در آن x، y و z به معنای نسخه اصلی، مینور و نسخه PHP است. برای نسخه های 32 بیتی PHP در سیستم عامل 64 بیتی، کلیدهای رجیستری زیر به ترتیب بررسی می شوند: [HKEY_LOCAL_MACHINE\SOFTWARE\WOW6421Node\PHP\x.y.z], [HKEY_LOCAL_MACHINE\SOFTWARE\WOW6421Node\PHP\x.y] و [HKEY_LOCAL_MACHINE\SOFTWARE\WOW6421Node\PHP\x], که در آن x، y و z به معنای نسخه اصلی، مینور و نسخه PHP است. اگر مقداری برای IniFilePath در هر یک از این کلیدها وجود داشته باشد، اولین مورد یافت شده به عنوان مکان php.ini استفاده خواهد شد. (فقط ویندوز).
  • [HKEY_LOCAL_MACHINE\SOFTWARE\PHP] یا [HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\PHP], مقدار IniFilePath (Windows only).
  • پوشه کاری فعلی (به جز CLI).
  • پوشه وب سرور (برای ماژول های SAPI)، یا پوشه PHP (در غیر این صورت در ویندوز).
  • پوشه ویندوز (C:\windows یا C:\winnt) (برای ویندوز), یا --with-config-file-path گزینه زمان کامپایل

اگر php-SAPI.ini وجود داشته باشد (که SAPI در حال استفاده است، به عنوان مثال، php-cli.ini یا php-apache.ini)، به جای php.ini استفاده می شود. نام SAPI را می توان با php_sapi_name() تعیین کرد.

Note:

وب سرور آپاچی هنگام استارت‌آپ پوشه را به حالت تغییر می‌دهد و باعث می‌شود PHP سعی کند فایل php.ini را در صورت وجود بخواند.

استفاده از متغیرهای محیطی در php.ini همانطور که در زیر نشان داده شده است.

Example #1 php.ini متغیر محیطی

; PHP_MEMORY_LIMIT is taken from environment
memory_limit = ${PHP_MEMORY_LIMIT}

php.ini دستورالعمل هایی که توسط برنامه های افزودنی مدیریت می شوند در صفحات مربوطه خود برنامه های افزودنی مستند می شوند. لیست دستورالعمل های اصلی در پیوست موجود است. همه دستورالعمل‌های PHP لزوماً در این راهنما ثبت نمی‌شوند: برای فهرست کامل دستورالعمل‌های موجود در نسخه PHP، لطفاً php.ini خود را که به خوبی نظر داده شده است، بخوانید. فایل همچنین، می‌توانید » آخرین php.ini را از Git بیابید. نیز مفید است.

Example #2 php.ini مثال

; هر متنی در یک خط پس از نقطه ویرگول (;) نقل قول نشده نادیده گرفته می شود
[php] ; نشانگرهای بخش (متن داخل پرانتز) نیز نادیده گرفته می شوند
; مقادیر بولی را می توان به یکی از موارد زیر تنظیم کرد:
;    true, on, yes
; or false, off, no, none
register_globals = off
track_errors = yes

;می توانید رشته ها را در دو گیومه قرار دهید
شامل_مسیر = ".:/usr/local/lib/php"

; با بک‌اسلش مانند هر کاراکتر دیگری رفتار می شود
شامل_مسیر = ".;c:\php\lib"

می توان به متغیرهای .ini موجود در فایل های .ini اشاره کرد مثال : open_basedir = ${open_basedir} ":/new/dir".

اسکن پوشه ها

پس از خواندن php.ini امکان پیکربندی فایل های PHP برای اسکن فایل‌های .ini وجود دارد و در زمان کامپایل می‌توان آن را تنظیم کرد. سپس با تنظیم متغیر محیطی PHP_INI_SCAN_DIR، دایرکتوری اسکن را می توان در زمان اجرا لغو کرد.

اسکن چندین دایرکتوری با جدا کردن آنها با جداکننده مسیر مخصوص پلتفرم (; در Windows، NetWare و RISC OS امکانپذیر است; تمامی پلتفرم‌های دیگر؛ مقدار PHP است در تمامی پلتفرم‌های دیگر مقداری که PHP استفاده می کند به عنوان ثابت PATH_SEPARATOR در دسترس است. اگر پوشه خالی در PHP_INI_SCAN_DIR داده شود، PHP همچنین دایرکتوری داده شده در زمان کامپایل را از طریق --with-config-file-scan-dir اسکن می کند.

درون هر پوشه،‌ PHP تمام فایل ها با پسوند .ini را به ترتیب حروف الفبا اسکن می‌کند. لیستی از فایل هایی که بارگزاری شدند، ترتیب آنها با فراخوانی دستور php_ini_scanned_files() ،و یا با ران کردن PHP با گزینه --ini امکان‌پذیر است. 

با فرض پیکربرندی شدن PHP توسط --with-config-file-scan-dir=/etc/php.d,
و این آدرس جداکننده است : ...

$ php
  PHP تمام فایل های درون  /etc/php.d/*.ini را به عنوان فایل های پیکربندی بارگذاری می‌کند. 
  
$ PHP_INI_SCAN_DIR=/usr/local/etc/php.d php
  PHP تمام فای های در /usr/local/etc/php.d/*.ini را بع عنوان فایل های پیکربندی بارگذاری می‌کند.

$ PHP_INI_SCAN_DIR=:/usr/local/etc/php.d php
  PHP تمام فای های در /usr/local/etc/php.d/*.ini را بع عنوان فایل های پیکربندی بارگذاری می‌کند.

$ PHP_INI_SCAN_DIR=/usr/local/etc/php.d: php
  PHP تمام فای های در /usr/local/etc/php.d/*.ini را بع عنوان فایل های پیکربندی بارگذاری می‌کند.

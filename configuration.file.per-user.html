.user.ini فایل های

PHP پشتیبانی از پیکربندی فایل های INI را در پوشه بر اساس هر پوشه شامل می‌شود. این فایل تنها توسط CGI/FastCGI SAPI پردازش می‌شوند. این عملکرد اساس فهرست برنامه افزودنی PECL htscanner را منسوخ می کند. اگر شما PHP را بر روی ماژول APACHE اجرا می‌کنید، از فایل های .htaccess برای اثر مشابه استفاده کنید.

علاوه‌بر کل فایل php.ini PHP با شروع از پوشه‌ای که فایل PHP درخواست شده در آن وجود دارد تا به پوشه فعلی روت (همانطور که در $_SERVER['DOCUMENT_ROOT'] تنظیم شده است) برای تمام فایل ها با پسوند .ini اسکن می‌کند.

تنها فایل های INI با تنظیمات حالت PHP_INI_PERDIR و PHP_INI_USER توسط فایل های INI شناخته خواهندشد.

دو دستورعمل جدید INI، user_ini.filename و user_ini.cache_ttl کنترل فایل های INI کاربر را به عهده می‌گیرند.

user_ini.filename نام فایل مورد نظر PHP را در هر دایرکتوری تعیین می کند; اگر روی یک رشته خالی تنظیم شود، PHP اصلا اسکن نمی کند. را پیش فرض .user.ini است

user_ini.cache_ttl تعداد دفعات بازخوانی فایل های INI کاربر را کنترل می کند. پیش فرض 300 ثانیه (5 دقیقه) است.

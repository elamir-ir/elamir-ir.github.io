چگونه تنظیمات پیکربندی را تغییر دهیم

PHP را به عنوان ماژول APACHE اجرا می‌کنید

هتکام استفاده از PHP به عنوان ماژول APACHE، شما همچنان می‌توانید تنظیمات پیکربندی را با استفاده از دستورعمل ها در فایل های (مثلا httpd.conf) و .htaccess پیکربندی APACHE تغییر دهید. شما نیاز به تنظیم دسترسی "AllowOverride Options" یا "AllowOverride All" برای انجام آن دارید.

دستورعمل های متعددی در APACHE وجود دارند که به شما امکان تغییر پیکربندی PHP را در APACHE می‌دهند. برای لیستی از دستورعمل که PHP_INI_ALL، یا PHP_INI_PERDIR، یا PHP_INI_SYSTEM, هستند، نکاهی بر پیوست لیست دستورعمل های php.ini بندازید.

php_value نام مقدار

مقدار دستورعمل مشخص شده را تنظیم می‌کند. تنها با دستور های حالت PHP_INI_ALL و PHP_INI_PERDIR می‌تواند استفاده شود. برای پاک‌کردن مقدار تنظیم شده قبلی از none به جای مقدار استفاده کنید.

Note: از php_value برای تنظیم مقادیر بولین استفاده نکنید. php_flag (پایین را ببینید) باید یه جای آن استفاده شود.

php_flag نام روشن|خاموش

برای تنظیم پیکربندی دستورعمل استفاده می‌شود. تنها با دستورعمل های حالت PHP_INI_ALL و PHP_INI_PERDIR می‌تواند استفاده شود.

php_admin_value نام مقدار

مقدار دستورعمل مشخص را تنظیم می‌کند. این نمیتواند در فایل .htaccess استفاده شود. هیچ دستورعملی که با php_admin_value شده باشد نمی‌تواند توسط .htaccess یا ini_set() رونویسی شود. برای پاک‌کردن مقدار تنظیم شده قبلی از none به جای مقدار استفاده کنید.

php_admin_flag نام روشن|خاموش

برای تنظیم پیکربندی بولین دستورعمل استفاده می‌شود. این نمیتواند در فایل .htaccess استفاده شود. هیچ دستورعملی که با php_admin_value شده باشد نمی‌تواند توسط .htaccess یا ini_set() رونویسی شود. برای پاک‌کردن مقدار تنظیم شده قبلی از none به جای مقدار استفاده کنید

Example #1 مثال پیکربندی آپاچی

<IfModule mod_php5.c>
  php_value include_path ".:/usr/local/lib/php"
  php_admin_flag engine on
</IfModule>
<IfModule mod_php4.c>
  php_value include_path ".:/usr/local/lib/php"
  php_admin_flag engine on
</IfModule>

Caution

ثابت های PHP خازج از آن وجود ندارند. به طور مثال در httpd.conf شما نمی‌توانید از E_ALL یا E_NOTICE برای تنظیم گزارش ارور دستورعمل ها استفاده کنید زیرا آن ها هیچ معنایی نخواند داشت و به عنوان 0 ارزش گذاری می‌شوند. در عوض، از مقدار بیت‌ماسک مربوطه استفاده کنید. این ثابت ها می‌توانند در php.ini استفاده شوند.

تغییر پیکربندی PHP از ظریق ریجستری ویندوز

هنگام اجرای PHP در ویندوز، مقادیر پیکربندی را می توان بر اساس هر دایرکتوری با استفاده از رجیستری ویندوز تغییر داد. مقادیر پیکربندی در کلید ریجستری HKLM\SOFTWARE\PHP\Per Directory Values در کلیدهای فرعی مربوط به نام مسیر ها ذخیره می‌شوند. به عنوان مثال، مقادیر پیکربندی برای پوشه c:\inetpub\wwwroot در کلید HKLM\SOFTWARE\PHP\Per Directory Values\c\inetpub\wwwroot ذخیره می‌شود. تنظیمات برای آن پوشه برای اسکریپتی در هر پوشه یا پوشه فرعی اجرا می‌شود فعال خواهد شد. مقادیر زیر کلید باید دارای اسم پیکربندی دستورعمل PHP و مقدار رشته را داشته باشند. ثابت های PHP در مقادیر تجزیه نمی شوند. با این حال، فقط مقادیر پیکربندی قابل تغییر در PHP_INI_USER را می توان به این ترتیب تنظیم کرد، مقادیر PHP_INI_PERDIR نمی توانند، زیرا این مقادیر پیکربندی برای هر درخواست دوباره خوانده می شوند.

سایر رابط های PHP

بدون توجه به چگونگی اجرا PHP، شما می‌توانید مقادیر مشخصی را در زمان اجرای اسکریپت توسط ini_set() تغییر دهید. مستند آن را در صفحه ini_set() برای اطلاعات بیشتر ببینید.

اگر شما به دیدن لیست کامل تنظیمات پیکربندی با مقادیر فعلی شان بر روی کامپیوتر خود علاقه‌مند هستید،‌ می‌توانید تابع phpinfo() را اجرا کنید و نتایج صفحه را بازبینی کنید. شما هم‌چنین می‌توانید به مقادیر پیکربندی هر یک از دستورعمل ها به وسیله ini_get() یا get_cfg_var() دسترسی پیدا کنید.
